Build a production-ready, self-hosted deployment platform called "Sky-Hosting" - a Heroku/Vercel alternative that deploys both frontend and backend applications from GitHub URLs via API, returning live HTTPS URLs.

**CRITICAL REQUIREMENTS:**
- This must be a FULLY FUNCTIONAL production system - NO mockups, NO demo data, NO placeholder functionality
- Every endpoint must actually deploy real code and return REAL live URLs
- All admin features must work with real data and real user management
- The system must be deployable on Render.com or Railway.app (provide full deployment configuration for both)

**Core Architecture:**
- REST API that accepts GitHub repository URLs and deploys them
- API key authentication for bot/service integration
- Admin dashboard for managing users, deployments, and system settings
- Automatic SSL/TLS certificate provisioning for all deployed apps
- Support for Node.js, Python, static sites, and Docker-based deployments
- Real build logs and deployment status tracking

**API Endpoints (all fully functional):**
POST /api/v1/deploy - Accept GitHub URL, return deployment ID and live URL
GET /api/v1/deployments/:id - Get deployment status and logs
GET /api/v1/deployments - List all deployments for authenticated user
DELETE /api/v1/deployments/:id - Remove deployment
POST /api/v1/auth/login - User/admin authentication
GET /api/v1/admin/users - List all users (admin only)
POST /api/v1/admin/users - Create user with API key (admin only)
DELETE /api/v1/admin/users/:id - Revoke user access (admin only)
GET /api/v1/admin/stats - System statistics (deployments, resource usage)

**Admin Dashboard Features:**
- User management: Create, disable, delete user accounts
- API key generation and revocation per user
- View all deployments across all users
- System resource monitoring (CPU, memory, disk)
- Deployment logs viewer with real-time updates
- Global settings: Domain configuration, resource limits, default buildpacks

**Deployment Engine:**
- Clone GitHub repository (public or private with access token)
- Auto-detect framework (Node.js, Python, Go, static)
- Run build process in isolated environment
- Deploy to appropriate runtime (container or static hosting)
- Assign unique subdomain: https://[app-name]-[random].sky-hosting.com
- Provision SSL certificate automatically
- Support custom domains via admin panel
- Environment variable management per deployment

**Technical Stack (choose one consistent approach):**
Option A - Use Dokploy (open-source Heroku alternative) as base and wrap with custom API
